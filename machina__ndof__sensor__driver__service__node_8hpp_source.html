62 /**
63  * @brief LifecycleNode to provide service to get latest sensor data
64  *
65  * Uses the @ref machina::tcp_interface::TcpInterface to read sensor data and
66  * advertise a service to serve the latest sensor data.
67  * A thread is spawned to read sensor data continuously and store it. When a
68  * service request is received, the latest sensor data is served.
69  * Since this is concurrent, we have defined 2 modes of operation:
70  * - Lock-free using TACO from
71  * iceoryx (https://iceoryx.io/latest/advanced/iceoryx_hoofs/#concurrent)
72  * - Using shared_mutex
73  * You can toggle between these modes by setting -DENABLE_LOCKFREE to ON/OFF
74  * ON for lock-free and OFF for shared_mutex. More experiments and data needs
75  * to be collected but right now TACO enables a significant speedup but it
76  * blocks access to the data often at high frequencies. Thinking of swapping to
77  * a 2 element container in the future so that the producer can always write to
78  * the 2nd element. If the 2nd element is being written to (can use atomic to
79  * check) then the consumer can read from the 1st element. This way we can
80  * avoid blocking. This is a future update.
81  *
82  * The node also uses the Lifecycle Design Pattern
83  * (https://design.ros2.org/articles/node_lifecycle.html)
84  * That means the node won't start just by the typical ros2 run routine. You
85  * need to call the lifecycle transitions configure and activate to enter
86  * nominal operation. Refer to the documentation for more details.
87  * https://github.com/ros2/demos/blob/iron/lifecycle/README.rst
88  *
89  * The node also uses the Composition Design Pattern
90  * (https://docs.ros.org/en/iron/Concepts/Intermediate/About-Composition.html)
91  * Ideally, this would mean there is a zero copy transfer of data between nodes
92  * in the same component container. However, I am not sure if that applies in
93  * the case of service-client relationship.
94  *
95  * A future update to be made here is to use TypeAdapters to reduce the
96  * number of memory copies. If we have 2 nodes in the same container and
97  * use type adapters, we could get the data, send it in the exact same format
98  * to the other node and it can use it directly. This would be a zero copy
99  *
100  */
