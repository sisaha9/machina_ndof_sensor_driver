34 /**
35  * @brief Class to handle the client side of the Machina NDof Sensor Driver
36  *
37  * This class provides an interface to the Machina NDof Sensor Driver to
38  * request sensor data and publish it for further processing.
39  *
40  * This class spawns multiple instances of @ref MachinaNDofSensorDriverClient
41  * handle multiple sensors. It periodically polls each client for the latest
42  * sensor data and publishes it.
43  *
44  * The class specifies it's own callback group for the timer so that it can
45  * run in parallel to the client-service callbacks in each instance of the
46  * @ref MachinaNDofSensorDriverClient class.
47  *
48  * The node also uses the Lifecycle Design Pattern
49  * (https://design.ros2.org/articles/node_lifecycle.html)
50  * That means the node won't start just by the typical ros2 run routine. You
51  * need to call the lifecycle transitions configure and activate to enter
52  * nominal operation. Refer to the documentation for more details.
53  * https://github.com/ros2/demos/blob/iron/lifecycle/README.rst
54  *
55  * The node also uses the Composition Design Pattern
56  * (https://docs.ros.org/en/iron/Concepts/Intermediate/About-Composition.html)
57  * Ideally, this would mean there is a zero copy transfer of data between nodes
58  * in the same component container. However, I am not sure if that applies in
59  * the case of service-client relationship. But for anyone subscribing to the
60  * output of this node, it should be a zero copy transfer.
61  */
